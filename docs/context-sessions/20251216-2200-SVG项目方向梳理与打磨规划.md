## 项目定位（已明确）

### 一句话描述
> **SVG 资产清洗器 + Sprite 生成器**
> 把零散的 SVG 文件批量导入 → 自动清洗/隔离 → 组织管理 → 导出成前端可用的 sprite 资产包

### 解决的痛点
| 痛点 | 方案 |
|------|------|
| 设计师给的 SVG 乱七八糟（带脚本、外链、复杂CSS） | 自动清洗 + 安全隔离 |
| 多个 SVG 的 id/class 会冲突 | 命名空间前缀隔离 |
| 想做可换色图标，但 SVG 里写死了颜色 | currentColor 一键规范化 |
| 手动拼 sprite.svg 太麻烦 | 自动生成 sprite + manifest |

### 目标用户
- 需要整理 SVG 图标库的前端开发者
- 想把 Figma/Sketch 导出的 SVG 批量处理的设计师
- 维护组件库图标资产的团队

### 竞品差异
| 工具 | 定位 | 本项目差异 |
|------|------|---------|
| SVGOMG | SVG 压缩优化 | 有资产管理 + sprite 生成 |
| IcoMoon | 图标字体生成 | 专注 SVG sprite，不做字体 |
| Figma | 设计工具 | 专注"清洗+导出"，更轻量 |
| svg-sprite (CLI) | 命令行工具 | 有可视化界面 + 预览 |
